# hostveil documentation

This directory holds contributor- and operator-facing
documentation that does not belong in the top-level `README.md`
(which is the user-facing entry point).

## Read in this order

1. **[README.md](../README.md)**: what hostveil is, how to
   install it, the user-facing commands and TUI controls.
2. **[ARCHITECTURE.md](ARCHITECTURE.md)**: package layout, data
   flow, scoring model, concurrency boundaries. Read this if
   you are changing the scoring engine, the fix engine, or
   the Web/TUI snapshot flow.
3. **[DEVELOPMENT.md](DEVELOPMENT.md)**: local workflow, build
   commands, test layout, conventions. Read this if you are
   about to commit code.
4. **[CONTRIBUTING.md](CONTRIBUTING.md)**: how to file issues,
   how to submit a PR, how to add a new fix rule.
5. **[SECURITY.md](../SECURITY.md)**: threat model, what
   hostveil protects against, how to report a vulnerability.
6. **[CHANGELOG.md](../CHANGELOG.md)**: release notes,
   breaking changes, security fixes.
